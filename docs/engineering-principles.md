# Engineering Principles

## Product

- Build for yourself first.
- Solve real problems.
- Small releases beat perfect releases.

## Engineering

- Keep the architecture simple.
- Refactor when duplication appears.
- Test before release.
- Document important decisions.

## User Experience

- Dashboard before complexity.
- Mobile first.
- Fast and intuitive interactions.
- Consistent UI across the ecosystem.

## Mindset

Measure.
Learn.
Improve.
Repeat.
